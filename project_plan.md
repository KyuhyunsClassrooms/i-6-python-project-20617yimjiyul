# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 100층의 탑

예) 기분과 시간에 따른 활동 추천기

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

예) 사용자가 현재 기분과 사용할 수 있는 시간을 입력하면, 조건에 맞는 활동을 추천해 주고 싶다.

내 설명: 몬스터를 처치한다


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 플레이어 이름
- 전투 중 선택할 행동(공격, 스킬, 포션 사용 등)
- 보상 선택(회복, 포션, 검 선택)

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 플레이어와 몬스터가 번갈아 공격한다.
- 몬스터를 처치하면 경험치를 얻고 레벨업한다.
- 층을 클리어하면 보상을 받고 다음 층으로 이동한다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 전투 결과와 몬스터 상태
- 플레이어의 레벨, 체력, 장비, 현재 층

## 6. 사용할 제어구조

### 조건문을 사용할 부분

예) 사용자의 예산보다 가격이 낮은 항목만 추천한다.

내 계획:현재 층에 따라 다른 몬스터를 등장시킨다.
플레이어 체력이 0 이하가 되면 사망 처리한다.
10층마다 보스를 등장시킨다.


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획:게임이 끝날 때까지 반복한다.
전투가 끝날 때까지 공격을 반복한다.
몬스터 목록을 탐색하여 현재 층에 맞는 몬스터를 찾는다.
